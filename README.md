# design-pattern
设计模式
## 面向对象设计原则
1. 对接口编程，而不是对实现编程
2. 优先使用对象组合而不是继承
## 设计模式六大原则
1. 开闭原则：对扩展开发，对修改关闭
2. 里式代换原则：里式代换是对开闭原则的补充，规定在任何基类出现的地方，子类都可以出现。
3. 依赖倒转原则：是开闭原则的基础，针对接口编程，依赖于抽象而不依赖于具体
4. 接口隔离原则：降低依赖，降低耦合，使用多个隔离的接口比使用单个接口好。
5. 最少知道原则（迪米特法则）：一个实体应尽量少于其它实体之间发生相互作用，使得系统模块相互独立。
6. 合成复用原则：尽量使用合成聚合的方式，而不是继承。
